A central, sortable table of your git projects. =M-x project-overview=
opens a =tabulated-list-mode= buffer with one row per auto-discovered git
repository, surfacing the kind of status you usually have to dig for:

#+begin_example
Project                   Version   ChangeLog     Bugs   Branch   Git   Commit
ollama-buddy              7.5.3     2026-05-19    2/15   main     *      2026-06-02 14:32
simply-annotate           1.2.0     2026-05-31    4/9    develop  ↑2     2026-05-31 09:18
bank-buddy                0.3.1     2026-01-16            main           2026-01-16 21:05
#+end_example

The table opens sorted by =Commit= date/time, most recently committed first.

* What each column means

- *Version / ChangeLog* :: parsed from the first =** <date> *X.Y.Z*= heading in
  the project's =CHANGELOG.org=.
- *Bugs* :: =open/total= count from =BUGS.org= (=TODO=/=DOING= vs everything,
  including =DONE=). Highlighted when there are open items.
- *Branch / Git* :: current branch, =*= when the worktree is dirty, and
  =↑n=/=↓n= for commits ahead of / behind the upstream.
- *Commit* :: date and time of the last commit (=YYYY-MM-DD HH:MM=), giving
  a granular default sort.
- *Description* :: the first sentence of the project's =README.org= (or
  =README.md=), collapsed to a single line.

Click a column header (or press =S=) to sort by it.

A header line shows aggregate status across all projects, e.g.:

#+begin_example
 42 projects · 5 dirty · 3 out of sync · 28 open bugs in 9
#+end_example

The counts cover the full set regardless of any active filter; positive
counts are highlighted so problem areas stand out at a glance.

* Actions

Single keys act on the project under point.  The =project.el= actions use
the same keys as the =project-switch-project= menu:

| Key       | Action                                       |
|-----------+----------------------------------------------|
| =RET= / =o= | switch to project (=project-switch-project=) |
| =f=         | find file (=project-find-file=)              |
| =G=         | find regexp (=project-find-regexp=)          |
| =d=         | find directory (=project-find-dir=)          |
| =v=         | =project-vc-dir=                             |
| =e=         | =project-eshell=                            |
| =s=         | search (=consult-ripgrep=, else project grep)|
| =m=         | =magit-status=                               |
| =D=         | dired at root                                |
| =!=         | shell at root                                |
| =c=         | preview latest =CHANGELOG.org= entry          |
| =C=         | open =CHANGELOG.org= in a window to the right  |
| =b=         | open =BUGS.org= in a window to the right      |
| =B=         | TODO agenda scoped to this project's bugs    |
| =A=         | TODO agenda for every project's bugs         |
| =/=         | filter the view (transient)                  |
| =g=         | refresh (re-scan)                            |
| =?=         | transient menu of all the above              |

* Filtering

Press =/= (=project-overview-filter-dispatch=) to narrow the table to a
different view of your repositories:

| Key | View                                            |
|-----+-------------------------------------------------|
| =d= | only repos with uncommitted changes             |
| =b= | only repos with open =BUGS.org= entries          |
| =u= | only repos ahead of / behind their upstream     |
| =n= | only repos whose name matches a regexp          |
| =a= | clear the filter (show all again)               |

The active filter and a =shown/total= count appear in the mode line, e.g.
=[dirty 5/42]=.  Filters are remembered across =g= (refresh), so you can
keep, say, the "dirty" view live while you work through it, and clear it
with =/ a= when done.

Prefer a menu? =?= (=project-overview-dispatch=) opens a transient with the
same actions grouped into columns, headed by the project under point.

* Installation

#+begin_src emacs-lisp
(use-package project-overview
  :ensure nil
  :load-path "~/.emacs.d/offline-packages/local-packages/project-overview"
  :commands (project-overview)
  :custom
  (project-overview-search-roots
   (list "~/source/repos" "~/.emacs.d"))
  ;; Hide noise such as kernel source trees.
  (project-overview-exclude-regexp "\\`\\(linux-\\|melpa\\|themes\\'\\)"))
#+end_src

* Customization

- =project-overview-search-roots= :: directories scanned for git projects.
  Each root is checked directly and one level deep for =.git= subdirs.
- =project-overview-exclude-regexp= :: project directory names matching this
  regexp are hidden.
- =project-overview-buffer-name= :: name of the dashboard buffer.

* Notes

The =CHANGELOG.org= and =BUGS.org= parsing assumes the common org layout:

#+begin_example
#+todo: TODO DOING | DONE
* Versions
** <2026-05-19 Tue> *7.5.3*
#+end_example

but degrades gracefully when a file is missing or differently shaped.
