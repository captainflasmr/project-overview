#+TITLE: Changelog for project-overview

* [0.5.0] - 2026-06-10
** Added
- Incremental (asynchronous) scanning: the dashboard now draws immediately and refreshes each project's git/CHANGELOG/BUGS data a batch at a time on a short timer, so Emacs stays responsive while a large set of repositories is scanned.  Rows reuse the previous scan's data until each project is re-probed, and a ~scanning N/M~ indicator appears in the header line.  Controlled by ~project-overview-async-scan~ (default on) and ~project-overview-async-batch-size~ (default 4); set the former to nil for the old single-pass synchronous scan.
- ~.~ (~project-overview-show-detail~): open a detail "card" for the project under point in a window to the right — an Org buffer summarising its status (version, branch and git state, remote, bug counts, GitHub issue/PR counts when fetched, and description) plus the open bug headings and the ten most recent commits.  Reuses the scanned cache, so it is near-instant.

* [0.4.0] - 2026-06-10
** Added
- ~Last Commit~ column: the subject (first line) of the most recent commit message, sitting alongside the existing ~Commit~ date column.  Included in the ~full~ and ~status~ views.
** Changed
- Tightened column widths across the board to fit more on screen: the single-character ~Known~ and ~MELPA~ columns are now headed ~K~ and ~M~, and ~Version~, ~ChangeLog~, ~Bugs~, ~Branch~, ~Git~, ~Remote~, ~Owner~, and ~GitHub~ were each trimmed to hug their content.
- The filter transient now clears the active filter with ~/ /~ instead of ~/ a~.

* [0.3.0] - 2026-06-05
** Added
- ~GitHub~ column: open issue and PR counts (e.g. ~3i 1p~) for GitHub-hosted repositories, fetched asynchronously via the ~gh~ CLI so the dashboard stays responsive and rows fill in once the queries return.  Requires ~gh~ to be installed and authenticated; toggle with ~project-overview-show-github~.
- ~i~ (~project-overview-github-issues~) and ~P~ (~project-overview-github-prs~): open the project's open GitHub issues or pull requests in an Org buffer — one heading per item with author, creation date, labels, URL, and the body — for a quick look at what they are.
- ~MELPA~ column: ~✓~ when a package matching the project's directory name is available from MELPA, checked locally against ~package-archive-contents~.  When no local MELPA contents are present, falls back (if ~project-overview-melpa-fallback~ is on) to fetching the package list from melpa.org once per session, asynchronously, so the column fills in shortly after.
- Persistent caching of network data: the MELPA package list and per-repository GitHub issue/PR counts are now cached on disk (~project-overview-cache-file~) with a ~project-overview-cache-ttl~ lifetime (default 1 day), surviving restarts so an ordinary refresh (~g~) reuses fresh data instead of re-hitting the network.
- ~r~ (~project-overview-cache-dispatch~): a transient to force-pull fresh GitHub counts, the MELPA list, or everything, and to clear the on-disk cache.
- New filter ~/ o~ (~project-overview-filter-owned~): show only repositories owned by ~project-overview-github-user~.
- New filter ~/ m~ (~project-overview-filter-melpa~): show only repositories whose package is available on MELPA.
- ~R~ (~project-overview-readme~): open the README (~.org~/~.md~/~.rst~/~.txt~/plain) of the project under point in a window to the right.
- ~TAB~ / ~S-TAB~ now move to the next / previous column (more ergonomic than the inherited ~M-left~/~M-right~).
- ~w~ (~project-overview-browse-remote~): open the project's remote repository (GitHub, GitLab, …) in a web browser, configurable via ~project-overview-browse-url-function~ (defaults to ~browse-url~; set to e.g. ~browse-url-firefox~ to always use Firefox).
- ~t~ (~project-overview-toggle-description~): quickly hide or show the ~Description~ column on top of any view (it is added to or removed from the active layout, not just views that already list it), without switching layouts.  The column is hidden by default now; set ~project-overview-show-description~ to show it on open.
- ~Path~ column: the project's root directory (abbreviated, e.g. ~~/source/repos/foo~).
- Selectable column layouts (views): ~V~ (~project-overview-cycle-view~) steps through ~full~ (every column), ~minimal~, ~status~ (branch/git/bugs), and ~remote~ (forge/owner) one press at a time, wrapping round (a prefix argument cycles backwards).  Layouts are defined in ~project-overview-views~ and the opening layout in ~project-overview-default-view~.
- The view chosen with ~V~ is remembered across sessions via ~savehist~ (~project-overview-saved-view~), taking precedence over ~project-overview-default-view~; toggle with ~project-overview-remember-view~.
** Changed
- Header line: now compact — leads with the active view (~[view]~), then the project total and only the non-zero counts (dirty, out of sync, bugs, and the open GitHub issues/PRs across ~project-overview-github-user~'s repos), with short labels; zero counts are dropped.
- Columns are now data-driven (~project-overview--columns~): the active view selects which appear and in what order, keeping the header and rows in sync.
- The ~?~ dispatch now lays the ~Dashboard~ group on the same row as ~Find~, ~Inspect~, and ~VC & search~; ~V~ (view) was added to it.

* [0.2.0] - 2026-06-04
** Added
- Combined bugs agenda: ~project-overview-bugs-agenda-all~ (~A~) gathers every discovered project's ~BUGS.org~ into a single TODO agenda, each entry categorised by its project; redo (~g~) keeps the same file scope and column alignment.
- Project actions mirroring the ~project-switch-project~ menu, acting on the project under point: ~G~ find regexp, ~d~ find directory, ~v~ ~vc-dir~, and ~e~ ~eshell~ (alongside the existing ~f~ find file and ~s~ search).
- Filtering (~/~, ~project-overview-filter-dispatch~): narrow the table to dirty repos, repos with open bugs, repos out of sync with upstream, or a name regexp; clear with ~/ a~.  The active filter is shown in the header line and mode line and survives a refresh.
- Header line summarising the whole set: total projects, and how many are dirty, out of sync, and have open bugs, with bold emphasis on non-zero values.
- Mode line shows the full (abbreviated) path of the project under point.
- ~Description~ column: the first sentence of the project's ~README.org~ or ~README.md~.
- ~Remote~ column: the forge hosting ~origin~ (~github~, ~gitlab~, ~codeberg~, ~bitbucket~, ~sourcehut~, or the bare host); blank for local-only repos.
- ~Owner~ column: the owner/user segment of the remote URL (e.g. the GitHub username or organisation).
- ~Known~ column: ~✓~ when the project is on Emacs's known-projects list.
- ~project-overview-include-known-projects~ (default on): merge ~project-known-project-roots~ (the ~project-switch-project~ list) into discovery alongside ~project-overview-search-roots~; non-git projects show blank git columns.
** Changed
- ~b~ (~BUGS.org~) and ~C~ (~CHANGELOG.org~) now open in a window to the right; ~c~ previews the latest changelog entry (word-wrapped) in a side window.
- Default sort is now the last commit date/time, newest first, and the commit column shows ~YYYY-MM-DD HH:MM~ for a more granular sort.
- Renamed the ~Changed~ column to ~ChangeLog~.
- The header line uses the theme's ~header-line~ face throughout, for readable contrast on any theme.
- Point starts on the first project row when the dashboard opens.
- ~g~ now refreshes (re-scan) and ~G~ runs find regexp.
** Fixed
- The single-project (~B~) and all-projects (~A~) bugs agendas no longer show identical contents when ~org-agenda-sticky~ is enabled; each now regenerates against its own file set.

* [0.1.0] - 2026-06-03
** Added
- ~project-overview~: a sortable ~tabulated-list-mode~ dashboard with one row per auto-discovered git project under ~project-overview-search-roots~ (each root and its immediate git subdirs, minus ~project-overview-exclude-regexp~).  Columns show the project name, latest ~CHANGELOG.org~ version and date, open/total ~BUGS.org~ count (~TODO~/~DOING~ vs all), git branch, a dirty/ahead-behind flag, and the last commit date.
- Single-key actions on the project under point: ~RET~/~o~ switch (~project-switch-project~), ~f~ find file, ~m~ ~magit-status~, ~s~ search (~consult-ripgrep~ when available, else project grep), ~d~ dired, ~!~ shell, ~c~ open ~CHANGELOG.org~, ~v~ preview the latest changelog entry in a side window, ~b~ open ~BUGS.org~, ~B~ a TODO agenda scoped to that project's bugs, and ~g~ refresh.
- ~project-overview-dispatch~ (bound to ~?~): a ~transient~ menu presenting the same actions grouped into columns, headed by the project under point.
- Customisation: ~project-overview-search-roots~, ~project-overview-exclude-regexp~ and ~project-overview-buffer-name~.
